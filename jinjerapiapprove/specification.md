# ジンジャー ワークフロー自動分析システム 仕様書

## 1. システム概要
本システムは、ジンジャー（jinjer）ワークフローの進行中の申請データを取得し、OpenAIのGPT-4oを用いて自動分析・判定を行うツールです。
各申請の内容をAIが要約し、「OK / 要確認 / NG」の判定と、客観的な理由を生成します。

## 2. 主な機能
- **自動ログイン**: APIキーまたは既存のトークンを使用して認証を行います。
- **申請取得**: 進行中のワークフロー申請を自動で巡回し取得します。
- **AI分析**: 各申請の詳細、承認履歴、カスタム項目を含めてGPT-4oが分析します。
- **承認者フィルタリング**: 自身の社員番号を設定することで、自分が承認すべきタスクのみを抽出できます。
- **レポート出力**: 分析結果をCSVおよびExcel形式で出力します。Excelには申請詳細画面へのハイパーリンクが自動挿入されます。

## 3. セットアップマニュアル
他のPCや環境で本システムを動かすための手順です。

### ステップ1: Pythonのインストール
Python 3.8以上が必要です。インストールされていない場合は公式サイトからダウンロードしてください。
- [Python ダウンロード](https://www.python.org/downloads/)
- ※インストール時、**"Add Python to PATH"** にチェックを入れてください。

### ステップ2: コードの準備
本システムのプロジェクトフォルダを任意の場所にコピーします。

### ステップ3: ライブラリのインストール
コマンドプロンプト（またはPowerShell）を開き、プロジェクトフォルダ内で以下のコマンドを実行します。

```powershell
pip install -r requirements.txt
```

### ステップ4: 環境変数の設定
システムを実行するために、以下の情報を環境変数として設定してください。

| 変数名 | 内容 | 備考 |
| :--- | :--- | :--- |
| `JINJER_API_KEY` | ジンジャーAPIキー | トークン取得に必要 |
| `JINJER_API_SECRET` | ジンジャーAPIシークレット | トークン取得に必要 |
| `JINJER_COMPANY_CODE` | 企業コード | 必須 |
| `OPENAI_API_KEY` | OpenAI APIキー | 分析に必要 |
| `JINJER_MY_EMPLOYEE_ID` | 自身の社員番号 | 自分のタスクのみ抽出する場合に設定 |

---

## 4. 実行方法
以下のコマンドでWebアプリケーションを起動します。

```powershell
streamlit run app.py
```

実行すると、ブラウザが自動的に開き（通常は `http://localhost:8501`）、分析画面が表示されます。

1. **設定**: 左側のサイドバーで、職務に必要な情報やフィルタ条件（社員番号など）を入力します。
2. **開始**: 「分析を開始する」ボタンをクリックします。
3. **確認**: ブラウザ上で進行状況と、エクセルと同じ項目（判定、理由、要約等）のテーブルが表示されます。
4. **ダウンロード**: 「Excel形式でダウンロード」ボタンから、ハイパーリンク付きの分析レポートを保存できます。

## 5. 出力フォーマット
出力されるファイルには以下の項目が含まれます。

1. **申請書No.**: ジンジャー内の一意なID
2. **件名**: 申請のタイトル
3. **申請者**: 氏名
4. **申請日時**: 日付
5. **判定**: AIによる判定（OK / 要確認 / NG）
6. **理由**: AIが判断した具体的な根拠
7. **要約**: 申請内容の要点
8. **URL**: ジンジャーの申請詳細画面への直接リンク
