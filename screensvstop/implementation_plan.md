# スリープ抑止ツール (Screen Sleep Prevention Tool) 実装計画

ご提示いただいた設計思想（最小権限、透明性、公式APIの利用、エンタープライズ向けの監査・管理機能など）に基づき、Windows 向けのデスクトップアプリケーションの実装計画をご提案します。

## 1. 提案する技術スタック

*   **言語 / フレームワーク**: C# / .NET 8.0 (Windows 固有機能を使用するため `net8.0-windows` を指定)
*   **UI プラットフォーム**: Windows Forms 
    *   *理由*: タスクトレイ（NotifyIcon）常駐型の軽量なアプリケーションを作成するのに非常に適しており、不要なリソース消費（CPU/メモリ）を抑えられるため。
*   **使用する主要な Win32 API**:
    *   `SetThreadExecutionState` (`kernel32.dll`): スリープおよびディスプレイオフの抑止に利用。ユーザー権限で動作可能。
    *   `RegisterHotKey` / `UnregisterHotKey` (`user32.dll`): グローバルホットキーの登録用。

## 2. アプリケーションの基本アーキテクチャ

*   **常駐プロセス**: アプリケーション起動時はメインウィンドウを表示せず、タスクトレイ（システムトレイ）アイコンのみを表示するバックグラウンドプロセスとして動作。
*   **状態管理**: 「非抑止（デフォルト）」「一時抑止（タイマー）」「永続抑止」のステートマシンを持つ。
*   **イベント駆動・クリーンアップ**:
    *   アプリケーション終了時、パニック時、バッテリー低下時などのイベントをフックし、必ず `SetThreadExecutionState(ES_CONTINUOUS)` を呼び出して元の状態に復帰させる（クリーンアップの徹底）。
*   **ログ出力**: `EventLog` クラスを使用し、設定に応じて Windows のイベントビューアー（Application ログ等）に状態変更（誰がいつオン/オフにしたか）を記録。

## 3. 実装のフェーズ分けと機能案

### フェーズ 1: コア機能と基本UI (最小要件)
*   [NEW] `Program.cs` / `ApplicationContext`: タスクトレイアイコン専用のアプリケーションエントリポイント。
*   [NEW] `NativeMethods.cs`: `SetThreadExecutionState` の P/Invoke 定義。
*   [NEW] `SleepPreventer.cs`: 抑止状態の管理（オン/オフの切り替え、タイマー管理等）。
*   **UI**: トレイアイコンの右クリックメニュー（コンテキストメニュー）。
    *   有効にする (永続)
    *   有効にする (30分)
    *   有効にする (1時間)
    *   無効にする (デフォルト)
    *   終了
*   **視覚的フィードバック**: 抑止中と非抑止中でトレイアイコンの画像（色など）を変更。

### フェーズ 2: 高度なトリガーと安全性
*   **自動解除（バッテリー考慮）**: `SystemInformation.PowerStatus.BatteryLifePercent` と `SystemEvents.PowerModeChanged` を監視し、バッテリー駆動になった際やバッテリーが一定以下（例: 20%）になった際に自動で抑止をオフにする機能。
*   **イベントログ出力**: 状態が変化した際に Windows イベントログに書き込む機能（設定でオンオフ可能）。

### フェーズ 3: ユーザー設定機能とエンタープライズ対応
*   [NEW] `SettingsForm.cs`: ホットキー設定、スケジュール設定、ログ出力設定などを行う設定画面。
*   **設定の保存先**: レジストリ、または `appsettings.json`（グループポリシーによる一元管理を見据える場合はレジストリからの読み込みも対応可能にすると良い）。
