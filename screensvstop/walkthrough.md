# ScreenSvStop (スリープ抑止ツール) 完成報告

お待たせいたしました！設計思想に基づき、スリープ抑止アプリの実装およびビルドを完了させました。

> [!TIP]
> ユーザーの皆様に .NET 8 SDK のような追加インストールを強いることがないよう、**Windows に標準で組み込まれている .NET Framework コンパイラ (`csc.exe`) を使用する形に仕様を調整しました。** 
> これにより結果的に **単一の軽量な実行可能ファイル (.exe)** となっており、エンタープライズ環境や管理者権限がない環境でも即座に利用できる「最小構成かつ安全なデフォルト」を実現しています。

## 開発実績サマリー

*   **完全なポータブル化**: インストーラーや事前準備なしで動作する EXE ファイル (`ScreenSvStop.exe`) を構築。
*   **Win32 APIの採用**: `SetThreadExecutionState` を活用し、「ディスプレイ電源断」と「システムスリープ」をプロセスレベルで安全に抑止。
*   **UI要件をクリア**: 「バックグラウンドへの常駐」「トレイアイコンの動的生成」「コンテキストメニューによるON/OFFの即時切り替え」の実装。

## 使い方と機能

1.  ファイルエクスプローラーから `c:\antigravity\antigravity\screensvstop\ScreenSvStop.exe` をダブルクリックして起動します。
2.  画面右下のタスクトレイ（システムトレイ）に「灰色」の四角アイコン（Zzz）が表示されます（これが「デフォルト動作：非抑止」の状態です）。
3.  アイコンを右クリックし、メニューから以下のアクションを選択できます。
    *   **抑止 (永続的)**: ずっと画面が暗くならないようにします。アイコンが「青色」に変わります。
    *   **抑止 (30分 / 1時間)**: 選択した時間が経過すると自動的にスリープ抑止が解除されます（元の設定に戻ります）。
    *   **抑止を解除 (デフォルト)**: スリープ抑止を中止します。

## プログラム終了時の安全性
プログラム終了時やパニック（異常終了）の際は、リソース（フラグ）が確実に解放され、Windowsが元々の電源設定（スリープ設定等）に基づいて動作するように安全対策（クリーンアップ処理の保証）を行っています。

## フェーズ2: 監査ログ機能と自動安全解除 (完了)
本フェーズではエンタープライズでのより強力な利用に向けて、以下の機能を実装・完了しました。

1. **ネットワーク境界への監査ログの追記**
   * 指定されたネットワーク共有フォルダ (`\\bellfilesv01.belluna.local\user1_iss\public\screecsvstop_log.txt`) へ「誰がいつどのような操作をしたか」を漏れなく記録します。
   * **ファイルロックのハンドリング**: 複数人が同タイミングで操作を行い、ファイルがロックされていてもエラー終了せずに、内部リトライ（待機と再試行）を行う機構を備えています。

2. **高度なトリガーと安全性機能 (自動解除)**
   * **バッテリー監視機構**: スリープ抑止中にデバイスが「バッテリー駆動」かつ「バッテリー残量20%以下」になると、バッテリー枯渇を防ぐために抑止を**自動解除**し、ユーザーに警告バルーンを表示します。
   * **画面ロック連動**: Windowsの機能で画面をロック（`Win + L`等）した場合も、放置状態での無駄な電源消費を防ぐため、スリープ抑止を**自動解除**し通常の電源ポリシーに従わせます。
   * これら自動解除された場合も、共有フォルダ上の監査ログに「自動解除された」旨が適切に記録されます。
